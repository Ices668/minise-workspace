Core Principles
关键原则
01

设计稿就是全部真相

不写功能设计文档，以 Stitch 输出的 HTML 设计稿为唯一参考，前端还原以设计稿为准。

02

每页只生成一次

generate_screen_from_text 每页只调用一次，遇到超时不重试，用 get_screen 轮询确认结果。

03

DESIGN.md 锁定风格

Boss 确认第一批设计稿后立即生成 DESIGN.md，后续所有页面生成必须注入设计系统 Token。

04

Boss 视觉验收即终点

不需要 Code Review 阶段，不需要后端，纯前端展示 Demo，Boss 确认外观即为交付通过。
